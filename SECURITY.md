# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| `main` | Yes, until the first stable release |
| Latest stable minor | Yes, after `1.0.0` |
| Prerelease tags | Best-effort follow-up prereleases |
| Older minors | No |

## Reporting a Vulnerability

Please report security issues privately rather than opening a public issue.

Preferred channels:

- GitHub Private Vulnerability Reporting: https://github.com/MatthiasReinholz/hungry-flamingo-blog-wordpress-theme/security/advisories/new
- Email: security@matthiasreinholz.com

Include reproduction steps, affected files, impacted versions, and expected impact. Expect an initial response within 3 business days and a severity/update assessment within 7 business days.

The maintainer coordinates fixes privately first, then publishes release notes and advisories after a patched release is available. Public disclosure timing depends on severity, exploitability, and user risk, with a target disclosure window of 30 days for confirmed vulnerabilities unless active exploitation requires faster publication.

Release artifacts may include provenance attestations, a CycloneDX SBOM, and keyless Sigstore bundles. Verify release assets before deploying security updates.
