# Editorial Workflow

Hungry Flamingo Blog is a block theme. Keep layout decisions in the Site Editor and keep durable site behavior in plugins.

## Templates

Use templates for structural page types: home, archives, single posts, pages, and WooCommerce store screens. Templates should answer what belongs on this type of page rather than what this single article should say.

## Template Parts

Use template parts for shared chrome: header, footer, and sidebar. Changes to a template part affect every template that includes it, so avoid campaign-specific content here.

## Unsynced Patterns

Use unsynced patterns for repeatable starting layouts. The theme includes editorial patterns and WooCommerce presentation patterns that can be inserted, edited, and then treated as normal page content.

## Synced Patterns

Use synced patterns only when every instance should stay identical. Good examples are persistent disclosure bands, store policy summaries, or a repeated site notice.

## WooCommerce Patterns

The WooCommerce patterns are presentation helpers. Product data, cart behavior, checkout, payments, orders, and customer records remain owned by WooCommerce. Insert these patterns only on sites where WooCommerce is active.

## Companion Plugin

Use the companion plugin for reader-retention behavior such as the continuous-reading stack, related posts, and editorial reports. The theme should remain installable without that plugin.
